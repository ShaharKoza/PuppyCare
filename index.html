Overview

What is PuppyCare?

PuppyCare is a dedicated iPhone app for continuous, real-time monitoring of your dog's environment. A Raspberry Pi sensor unit installed near the kennel collects data across 6 measurement types — temperature, humidity, light, motion, sound, and derived sleep state — and transmits it instantly to Firebase Realtime Database, which propagates every update to the iOS app in under a second.

The system is built around three core components: the Raspberry Pi sensor station running a Python script with the Firebase Admin SDK; the Firebase backend handling real-time sync and Cloud Function–powered push notifications; and a native SwiftUI iOS app (iOS 16+) that presents live sensor tiles, an alert history, feeding & routine management, an AI food-safety assistant, and a full dog profile — all in one coherent interface.

Alert thresholds are age- and breed-adjusted: puppies receive stricter monitoring than adults, and breeds like Brachycephalics or Nordic double-coated dogs have dedicated safety ranges backed by USDA APHIS, Purdue Canine Welfare, VCA Animal Hospitals, and Royal Canin Academy research. PuppyCare is not just a monitoring tool — it is a complete, evidence-based care platform.

Features

Everything your puppy needs.

Six sensors, four alert levels, four app tabs — PuppyCare covers the full spectrum of modern kennel care.

🌡️

Temperature & Humidity

DHT22 sensor polled every 5 s. Age-adjusted thresholds from newborn (29–32°C) to adult (20–29°C). Humidity monitoring 30–70% for adults, 55–65% for puppies.

🏃

Motion & Sleep Detection

PIR motion sensor with last-motion timestamp and seconds-since counter. Sleep state is derived from darkness + absence of motion + silence — not a single sensor.

🔊

Sound & Bark Analysis

Detects general sound and classifies barks separately. Unclassified sound does not count in the bark count. Triggers alerts after sustained barking bursts.

🚨

4-Level Alert System

Normal → Warning → Stress → Emergency. Cloud Functions trigger FCM push notifications instantly. Thresholds are breed- and age-specific per USDA APHIS guidelines.

📷

On-Demand Camera

Tap to request a live kennel snapshot. The Pi responds to a Firebase write, captures and uploads the image; the app displays it within seconds with a timestamp.

🍖

Care & Routine Management

Daily meal schedule with gram tracking and calorie totals. Walks and play sessions with countdowns. AI food-safety assistant + auto-generated vaccine reminders.

Alert System

Four levels of protection.

The system continuously evaluates all sensor data and assigns the kennel one of four severity states. Thresholds are age- and breed-adjusted — puppies receive stricter monitoring.

Normal

All Clear

All parameters are within safe ranges. No action required. The kennel environment is comfortable and stable.

🟢 Temperature, humidity, motion, and sound all nominal.
Warning

Monitor Closely

One parameter is approaching a threshold limit. Recommended to keep an eye on the situation — no immediate action needed yet.

🟡 Temperature reaching the upper comfort boundary.
Stress

Check Soon

Multiple signs of discomfort detected, or a single parameter is outside the comfort range. Check on your dog soon.

🟠 Prolonged barking + elevated temperature simultaneously.
Emergency

Immediate Action

A critical threshold has been crossed. An FCM push notification is dispatched instantly. Immediate response is required.

🔴 Temperature above 35°C or below 7°C (adult dog).
Young Puppy (0–4 weeks) 24–30°C safe Emergency: <20°C / >34°C
Older Puppy (1–6 months) 22–28°C safe Emergency: <18°C / >32°C
Adult Dog 18–26°C safe Emergency: Δ≥4°C from range
Humidity (all ages) 30–70% safe Stress: Δ≥15% from range
Bark Count (5 s window) 0–2 events = quiet ≥3 events = bark_detected alert
Sleep Detection Dark + quiet + still Requires 420 s inactivity

Thresholds from main_sensors.py v2 (Puppy-Safe Edition) · Sources: USDA APHIS · Purdue Canine Welfare · VCA Animal Hospitals · Royal Canin Academy · NIH/PMC

Technical Details

Under the hood.

A purpose-built stack that prioritises real-time performance, native iOS quality, and serverless scalability.

🏗️ App Architecture

  • UI Layer SwiftUI, Combine, @MainActor — Dashboard, Routine, Assistant & Profile tabs; all views reactive to @Published sensor state
  • Core Services FirebaseService · AlertManager · SensorHistoryStore · ProfileStore · FoodAssistantService · ReminderManager
  • Data Flow Pi → kennel/sensors (primary) + kennel/alerts (feed) → iOS listeners → UI re-render <1 s end-to-end
  • Pi Script — v2 Python · main_sensors.py Puppy-Safe Edition v2 · multi-threaded DHT reader · AlertEngine with puppy age tiers (young / older)

🔧 Raspberry Pi + Sensors (v2)

  • DHT22/AM2302 — GPIO 22 Temperature + humidity, polled every 10 s with sanity bounds & delta filtering. Written to kennel/dht and kennel/sensors
  • LDR — GPIO 4 Light/dark detection, polled every 2 s. Combined with motion + silence to derive sleep state. Written to kennel/light
  • KY-038 — GPIO 17 Sound + bark classification. 5 s sliding window, bark threshold = 3 events; sustained sound = 1.5 s continuous HIGH. Written to kennel/sound
  • PIR — GPIO 27 (active LOW) Motion detection with 5 s warmup on boot and 10 s holdoff. Sleep declared after 420 s of combined inactivity (dark + quiet + still). Written to kennel/pir

☁️ Firebase + Notifications

  • Primary path — kennel/sensors Flat combined snapshot written every alert cycle (~5 s) — iOS dashboard reads here: temperature, humidity, light, motion, sound, sleeping, streaks
  • Alert feed — kennel/alerts Push-appended list — each non-normal event appended with message, priority, and timestamp. iOS alert tab reads here
  • Detail paths kennel/dht · kennel/light · kennel/sound · kennel/pir · kennel/alert · kennel/camera
  • Camera Captures every 30 s (periodic) + on every PIR trigger. Written to kennel/camera by camera.py

User Guide

Getting started.

Everything you need to set up, configure, and use PuppyCare from day one.

1

Introduction & System Overview

PuppyCare requires an iPhone running iOS 16+, an active Raspberry Pi connected to power and Wi-Fi, and internet on both devices. Three core components: sensor unit, Firebase backend, iOS app.

2

First Launch & Initial Setup

Enter your dog's name and breed, date of birth (directly drives alert thresholds), and current weight. Confirm — the system calibrates and navigates to the Dashboard. The profile saves automatically.

3

Dashboard — Real-Time Monitoring

Live tiles for temperature, humidity, light, motion, sound, sleep state, and alert level. Tap the camera card to request an on-demand kennel snapshot. Data refreshes continuously via Firebase listeners.

4

Sensor Details & Data Displayed

Six sensors: temperature (age-adjusted), humidity, light (used in sleep detection), PIR motion (stabilises after 1 min), sound & bark count (separate classification), and derived sleep state.

5

Alert System & Severity Levels

Four levels: Normal → Warning → Stress → Emergency. Thresholds are adjusted to your dog's profile. Emergency events dispatch FCM push notifications instantly, even when the app is closed.

6

Dog Profile & Personalization

Keep name, breed, sex, date of birth, and weight accurate — inaccurate data affects the entire alert system. Profile data directly drives breed-specific thresholds and auto-generated vaccine reminders.

7

Troubleshooting & Common Issues

No data? Check Pi power, Wi-Fi, and the sensor script. 0°C reading? Check wiring. Motion always active? PIR needs 1 min to stabilise. Alerts wrong? Re-enter date of birth and puppy status.

Troubleshooting

Common issues & fixes.

Quick answers to the most frequent setup and runtime questions.

📡

No data on the Dashboard

  • Verify the Raspberry Pi is running and connected to power
  • Verify Wi-Fi connection on both the unit and phone
  • Verify the sensor script is active on the Pi
  • Check Firebase security rules allow read access
🌡️

Unexpected 0°C temperature reading

  • A 0°C reading is not necessarily an error — it can be a valid low-temp value
  • If abnormal over time, check the DHT22 sensor wiring
  • Restart the Pi unit if the issue persists
🏃

Motion sensor detecting continuous activity

  • Wait one minute after starting the Pi — PIR requires stabilisation time
  • Check if detection stabilises after a short period
  • If it continues, adjust PIR sensitivity settings
⚙️

Alerts too sensitive or not sensitive enough

  • Verify the date of birth was entered correctly — this directly drives thresholds
  • Verify puppy status is set correctly in Profile
  • Alert thresholds vary according to age and weight
🔊

No bark detection despite audible sound

  • Check if the sound is general noise not classified as a bark
  • Verify bark detection mechanism is active on the Pi
  • Adjust sound sensor sensitivity as needed
🔔

Not receiving push notifications

  • Grant push notification permission on first launch in iOS Settings
  • Confirm the Pi is writing to kennel/fcm_token correctly
  • Verify Firebase Cloud Functions are deployed and active
