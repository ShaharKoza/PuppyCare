Overview

What is PuppyCare?

PuppyCare is a dedicated iPhone app for continuous, real-time monitoring of your dog's environment. A Raspberry Pi sensor unit installed near the kennel collects data across 6 measurement types — temperature, humidity, motion, sound, light, and derived sleep state — and transmits it instantly to Firebase Realtime Database, which propagates every update to the iOS app in under a second.

The system is built around three core components: the Raspberry Pi sensor station running a Python script with the Firebase Admin SDK; the Firebase backend handling real-time sync and Cloud Function–powered push notifications; and a native SwiftUI iOS app (iOS 16+) that presents live sensor tiles, an alert history, feeding & routine management, an AI food-safety assistant, and a full dog profile — all in one coherent interface.

Alert thresholds are age- and breed-adjusted: puppies receive stricter monitoring than adults, and breeds like Brachycephalics or Nordic double-coated dogs have dedicated safety ranges backed by USDA APHIS, Purdue Canine Welfare, VCA Animal Hospitals, and Royal Canin Academy research. PuppyCare is not just a monitoring tool — it is a complete, evidence-based care platform.

Features

Everything your puppy needs.

Six measurements, three alert levels, four app tabs — PuppyCare covers the full spectrum of modern kennel care.

🌡️

Temperature & Humidity

DHT22/AM2302 — 3-wire (VCC / DATA / GND), GPIO 22. Polled every 10 s with retries, range validation (-10…60 °C), and 60 s warm cache. Age-adjusted thresholds: newborn 29–32°C → adult 20–29°C. Humidity: 55–65% (puppies) / 30–70% (adults).

🏃

Motion & Sleep Detection

HC-SR501 / AM312 PIR — Fresnel-lens dome, 3-wire (VCC / OUT / GND), GPIO 27, active HIGH. 5 s boot warm-up, 10 s holdoff per trigger. Sleep state derived on the Pi: dark + quiet + still for 420 s continuous.

🔊

Sound & Bark Analysis

KY-038 — electret microphone + LM393 comparator, GPIO 17. 5 s sliding window classifies events: ≥3 barks = bark_detected, ≥1.5 s continuous HIGH = sustained_sound (warning escalation). Sensitivity trimmer adjusts threshold.

💡

Ambient Light (LDR)

Photoresistor — 3-wire (VCC / OUT / GND), GPIO 4. Polled every 0.2 s with 3-sample debounce. HIGH = light, LOW = dark (polarity flag in code). Drives the dashboard Light tile and is logged as an informational event on each transition.

🚨

3-Tier Alert System

Bark → Critical immediately. Motion → Warning. Bark + Motion within 30 s → Critical (Activity + noise). Temperature out of range → Warning / Critical by severity. Cloud Function delivers FCM push the moment the level escalates.

📷

Live Camera Snapshots

The Pi captures from a USB / CSI camera every 30 s + on every PIR / bark event, publishes a cache-busted URL to kennel/camera, and the iOS app's AsyncImage reloads automatically. ATS allows local-network HTTP for the Pi.

🍖

Care & Routine Management

Daily meal schedule with gram tracking and calorie totals. Walks and play sessions with countdowns. AI food-safety assistant + auto-generated vaccine reminders by breed and age.

Alert System

Three tiers of protection.

The system continuously evaluates all sensor data and assigns the kennel one of three severity states. Thresholds are age- and breed-adjusted — puppies receive stricter monitoring. Bark events trigger Critical immediately.

Normal

All Clear

All parameters are within safe ranges. No action required. The kennel environment is comfortable and stable.

🟢 Temperature, humidity, motion, sound, and light all nominal.
Warning

Monitor Closely

Motion-only activity in the kennel without any noise — the puppy is moving but not distressed. Keep an eye on the situation.

🟡 Movement detected, no barking. Temperature reaching the comfort boundary.
Critical

Immediate Action

Any bark detected, or barking + motion together within a 30-second window, or a critical environmental threshold crossed. An FCM push notification is dispatched instantly.

Why PuppyCare matters.

Three core strengths that make PuppyCare more than a monitoring tool.

01
⚡

Real-Time Awareness

Sub-second data propagation from Pi to iPhone via Firebase. Critical alerts arrive as push notifications within seconds of a threshold breach — not minutes. Owners always know the kennel state, even at a distance.

02
📊

Better Puppy Care Decisions

The integrated analytics engine surfaces patterns invisible to casual observation: peak bark hours, activity cycles, temperature drift, and sustained sound events. Raw sensor data becomes actionable insight that improves long-term care quality.

🔗

Most IoT systems treat the mobile app as a passive display. PuppyCare treats it as a first-class platform: the iOS app initiates hardware actions, manages the notification pipeline, and stores the complete care record in one coherent interface.

Technical Details

Under the hood.

A purpose-built stack that prioritises real-time performance, native iOS quality, and serverless scalability.

🏗️ App Architecture

  • UI Layer SwiftUI, Combine, @MainActor — Dashboard, Routine, Assistant & Profile tabs; all views reactive to @Published sensor state
  • Core Services FirebaseService · AlertManager · SensorHistoryStore · ProfileStore · FoodAssistantService · ReminderManager
  • Data Flow Pi → kennel/sensors (primary) + kennel/alert (change-deduplicated) → iOS listeners → UI re-render <1 s end-to-end
  • Pi Script — v2 Python · main_sensors.py Puppy-Safe Edition v2 · multi-threaded DHT reader · AlertEngine with puppy age tiers (young / older)

🔧 Raspberry Pi + Sensors (v2)

  • DHT22/AM2302 — GPIO 22 Black plastic housing with ventilation grid · 3-wire (VCC / DATA / GND) · polled every 10 s with retries + 60 s warm cache + range validation (-10…60 °C) · writes kennel/sensors
  • KY-038 — GPIO 17 Red PCB · electret microphone + LM393 comparator · AO / G / + / DO pins · sensitivity trimmer · LED1 indicator · 5 s window, bark ≥ 3 events, sustained ≥ 1.5 s HIGH · writes kennel/sound
  • HC-SR501 PIR — GPIO 27 (active HIGH) White Fresnel-lens dome on square PCB · 3-wire (VCC / OUT / GND) · 5 s boot warmup · 10 s holdoff per trigger · sleep declared after 420 s combined inactivity · writes kennel/sensors
  • LDR Photoresistor — GPIO 4 Digital photoresistor module with onboard comparator · polled every 0.2 s · HIGH = light, LOW = dark · 3-sample debounce · writes kennel/sensors.light

☁️ Firebase + Notifications

  • Primary path — kennel/sensors Flat combined snapshot written every alert cycle (~5 s) — iOS dashboard reads here: temperature, humidity, motion, sound, sleeping, streaks
  • Alert state — kennel/alert Single node overwritten only when level or reasons change (deduplicated writes) — iOS alert tab derives its history from this stream client-side
  • Detail paths kennel/sensors · kennel/sound · kennel/alert · kennel/camera · kennel/diagnostics
  • Camera Captures every 30 s (periodic) + on every PIR trigger. Written to kennel/camera by camera.py

User Guide

Getting started.

Everything you need to set up, configure, and use PuppyCare from day one.

1

Introduction & System Overview

PuppyCare requires an iPhone running iOS 16+, an active Raspberry Pi connected to power and Wi-Fi, and internet on both devices. Three core components: sensor unit, Firebase backend, iOS app.

2

First Launch & Initial Setup

Enter your dog's name and breed, date of birth (directly drives alert thresholds), and current weight. Confirm — the system calibrates and navigates to the Dashboard. The profile saves automatically.

3

Dashboard — Real-Time Monitoring

Live tiles for temperature, humidity, motion, sound & bark count, ambient light (day/night), sleep state, and alert level. Tap the camera card to request an on-demand kennel snapshot. Data refreshes continuously via Firebase listeners.

4

Sensor Details & Data Displayed

Six measurements: temperature (age-adjusted), humidity, PIR motion (HC-SR501, active HIGH — requires a brief warm-up after boot), sound & bark count (separate classification), ambient light (LDR — day/night detection), and derived sleep state.

5

Alert System & Severity Levels

Three tiers: Normal → Warning (motion alone) → Critical (any bark, or bark + motion together within a 30-second window). Bark events fire Critical immediately. Thresholds adapt to your dog's profile. Critical events dispatch FCM push notifications instantly, even when the app is closed.

6

Dog Profile & Personalization

Keep name, breed, sex, date of birth, and weight accurate — inaccurate data affects the entire alert system. Profile data directly drives breed-specific thresholds and auto-generated vaccine reminders.

7

Troubleshooting & Common Issues

No data? Check Pi power, Wi-Fi, and the sensor script. 0°C reading? Check wiring. Motion stuck active? HC-SR501 PIR needs ~30s to stabilise. Light tile wrong? LDR digital module flips polarity — toggle LDR_ACTIVE_HIGH. Alerts wrong? Re-enter date of birth and puppy status.

Troubleshooting

Common issues & fixes.

Quick answers to the most frequent setup and runtime questions.

📡

No data on the Dashboard

  • Verify the Raspberry Pi is running and connected to power
  • Verify Wi-Fi connection on both the unit and phone
  • Verify the sensor script is active on the Pi
  • Check Firebase security rules allow read access
🌡️

Unexpected 0°C temperature reading

  • A 0°C reading is not necessarily an error — it can be a valid low-temp value
  • If abnormal over time, check the DHT22 sensor wiring
  • Restart the Pi unit if the issue persists
🏃

Motion sensor detecting continuous activity

  • Wait 30–60 seconds after starting the Pi — the AM312 PIR requires a brief warm-up period before readings stabilise
  • Confirm wiring: VCC → 3.3V, GND → GND, OUT → GPIO 27 (active HIGH)
  • Cover the lens fully — output should drop to 0 within ~3 seconds
  • If continuously HIGH, check for IR sources (sun, heaters) and reposition the unit
⚙️

Alerts too sensitive or not sensitive enough

  • Verify the date of birth was entered correctly — this directly drives thresholds
  • Verify puppy status is set correctly in Profile
  • Alert thresholds vary according to age and weight
🔊

No bark detection despite audible sound

  • Check if the sound is general noise not classified as a bark
  • Verify bark detection mechanism is active on the Pi
  • Adjust sound sensor sensitivity as needed
💡

Light tile shows wrong day/night state

  • Confirm wiring: VCC → 3.3V, GND → GND, DO → GPIO 4
  • Adjust the trim-pot on the LDR module until the on-board LED toggles at the desired brightness
  • If day/night are inverted, flip LDR_ACTIVE_HIGH in main_sensors.py
  • The reading is debounced over 3 samples — give it ~3 seconds to settle after a lighting change
🔔

Not receiving push notifications

  • Grant push notification permission on first launch in iOS Settings
  • Confirm the iOS app has registered an FCM token (check kennel/fcm_token in Firebase Console)
  • Verify Firebase Cloud Functions are deployed and active
